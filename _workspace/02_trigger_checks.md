# Trigger Checks

## should-trigger
- "Next.js 프론트와 API 백엔드를 동시에 진행할 하네스를 만들어줘."
- "와이어프레임부터 배포까지 웹사이트 팀 구조를 파일로 남겨줘."
- "디자인, 프론트, 백엔드, QA를 spawn_agent로 조율하는 플레이북을 구성해줘."
- "리더-워커 기반으로 풀스택 웹 개발 워커 체계를 만들어줘."

## should-not-trigger
- "지금 버튼 색만 바꿔줘."
- "이 컴포넌트 한 개만 리뷰해줘."
- "API 엔드포인트 하나 추가해줘."
- "배포 URL만 알려줘."

## 판단 기준
- 여러 역할이 필요한가
- 파이프라인이 단계형으로 이어지는가
- 재사용 가능한 역할/절차를 파일로 남겨야 하는가
- `_workspace/` 기반 핸드오프가 유용한가
