# QA Strategy

## 기본 선택
- QA 역할 포함: 예
- agent_type: `worker`
- 이유:
  - 테스트 실행과 스모크 체크가 필요하다
  - 프론트와 백엔드 경계면 비교가 필요하다
  - 배포 전 go / no-go 판단이 필요하다

## 점진적 QA 시점
1. 계약 문서 확정 직후
2. 프론트엔드와 백엔드 구현 직후
3. 통합 직후
4. 배포 인수인계 직전

## 핵심 검증 경계면
- API 응답 shape vs 프론트 기대 shape
- 성공 경로 vs 실패 경로
- 빈 상태 vs 데이터 존재 상태
- 요구사항 vs 실제 구현

## QA 산출물
- `_workspace/05_qa_release_report.md`

## 릴리즈 판단 기준
- blocker 없음
- 핵심 사용자 흐름 재현 가능
- 배포 전제조건과 env var 확인 완료
- 실행한 테스트/스모크 명령이 기록되어 있음
