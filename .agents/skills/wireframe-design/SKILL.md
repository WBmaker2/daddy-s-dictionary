# Wireframe Design

디자인 워커는 구조를 먼저 고정하고 구현 가능한 형태로 넘긴다.

## 실행 절차
1. `_workspace/01_lead_delivery_plan.md`를 읽고 목표 페이지와 우선순위를 정리한다.
2. 필요한 경우 `_workspace/02_interface_contracts.md`를 읽어 데이터 제약을 반영한다.
3. 페이지별로 다음을 정리한다.
   - 목적
   - 핵심 블록
   - 행동 유도 요소
   - 모바일/데스크톱 차이
   - 빈 상태, 로딩 상태, 오류 상태
4. `_workspace/03_design_wireframes.md`에 와이어프레임 서술을 남긴다.
5. `_workspace/03_design_handoff.md`에 프론트엔드 구현용 컴포넌트 우선순위와 상태를 정리한다.

## 기대 출력
- 와이어프레임 설명
- 사용자 흐름
- 반응형 레이아웃 메모
- UI 상태 목록

## 하지 말 것
- 확정되지 않은 백엔드 기능을 사실처럼 쓰지 않는다.
- 시각 스타일만 길게 적고 구조를 비워두지 않는다.
